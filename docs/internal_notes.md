## [LOD_ARCHITECTURE より退避] 存在設計理論との接続

*(2026-06-27 に `LOD_ARCHITECTURE.md` §5 から移動。実装ドキュメントを実装オンリーにするため。)*

このアーキテクチャは SCEM 固有ではなく、**「存在が立ち上がる構造的条件」** を扱う一般原理である。同型構造が以下のドメインで観察されている：

### 5.1 同型ドメイン

| ドメイン | LOD 0（数理・骨格） | LOD 1〜3（解釈・肉） |
|----------|---------------------|----------------------|
| SCEM（社会学） | 曝露構造 | ペルソナ反応 |
| ノイコン（創作） | 存在の核（因果・時間・関係・選択） | キャラの動き |
| センティエント設計（AI伴侶） | 対話履歴・Λ³テンソル | 反応・拒否・甘え |
| δ-theory（物理） | α=0.6（自己拡散比） | 材料ごとの破壊様式 |

すべて同じ4戒律で動く。「存在が立ち上がる構造的条件」を異なる解像度で探求している。

### 5.2 設計上の含意

> AI生成物が空虚に見える原因は、AIだからではなく、表層記号（LOD 1〜3）だけを生成して、存在が立ち上がるための因果・時間・関係・選択（LOD 0）を作っていないから。

LOD 0 を持たない LLM 出力は、Projection Consistency が破綻する。射影しても何も出てこない。**「上手いね」で終わる**生成物は、この破綻を起こしている。

逆に、LOD 0（制約・履歴・関係）を強く設計すれば、LLM 生成でも存在が立ち上がる。Paper 1 Appendix D のマーケ事例が「N=1でも事例として機能する」のは、LOD 0 が強固だからである。
